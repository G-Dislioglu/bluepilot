# BP-144 Review Packet - Write Permit contentHash Canon

## Scope

BP-144 implements stage 1 of the one-shot write permit design on the Bluepilot side:
a pure contentHash canonicalization helper and tests against the fixed shared vector.

## Changes

| File | Change |
|---|---|
| `contracts/BP-144.json` | WLP contract for the stage-1 Bluepilot implementation. |
| `builder/src/writePermitContentHash.ts` | Adds the pure canonical byte builder and SHA-256 contentHash helper. |
| `builder/tests/writePermitContentHash.test.ts` | Proves the shared vector, coordinate binding, and byte sensitivity. |
| `docs/SESSION-LOG.md`, `STATE.md` | Records the stage-1 canon as current Bluepilot context. |
| `review-packets/BP-144.md` | This review packet. |

## Safety Notes

- No permit register, consume operation, corridor logic, SmartPush path, provider path, route,
  runtime flag, dependency, or package script changed.
- The helper is side-effect-free and uses only `node:crypto` plus byte concatenation.
- The new helper name is deliberately not Opus-branded.
- Tests perform no network, DB, token, or GitHub write.
