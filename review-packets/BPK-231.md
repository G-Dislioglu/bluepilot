# BPK-231 Review Packet - Local Emergency Stop and Daily Provider Cap

## Scope

BPK-231 adds a small local safety guard for two entry points:

- Provider calls are checked before the remote Maya budget gate.
- `permitApply` writes are checked before `smartPush` through the existing builder safety path.

## Controls

- `BLUEPILOT_LOCAL_EMERGENCY_STOP=true|1|yes|on` blocks provider calls, permitApply writes, and future runtime targets locally.
- `BLUEPILOT_LOCAL_DAILY_PROVIDER_TOKEN_LIMIT=<positive integer>` blocks projected provider calls when today's process-local provider token count plus requested tokens would exceed the cap.

The daily cap is token-based, not USD-based. This avoids creating a second provider pricing table; Maya-core remains the canonical cost/budget source through `assessBudget` and `recordCost`.

## Safety Notes

- No new route, DB table, package file, workflow, deploy, provider pricing table, proof-gate, or runtime activation was added.
- `permitApply` still uses `guardBuilderPush`; local safety folds into the existing blocked decision path.
- Provider calls still use remote `assessBudget` and `recordCost`; local safety only adds a fail-closed precheck and process-local token counter.
