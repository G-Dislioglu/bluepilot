# BPK-002 Review Packet - Permit Generalisierung

## Summary

BPK-002 binds the generic `POST /probe/sandbox-write` endpoint back to the
existing Bluepilot write-permit corridor.

- Write requests now require a syntactically valid `permitId`.
- The endpoint inspects the target sandbox file state read-only, derives
  `create` or `update`, and passes `baseSha` into `smartPush(writePermit)`.
- `smartPush` remains the only consume/write corridor; no duplicate permit
  store or consume implementation was added.
- Delete/undo is blocked before GitHub mutation until a dedicated permit type
  exists.

No package files, server routing, maya-core code, deployment settings, runtime
environment values, or live GitHub writes were changed.
