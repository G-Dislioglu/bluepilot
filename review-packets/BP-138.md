# BP-138 Review Packet - Builder repo-index runtime artifact

## Summary

BP-138 supplies the missing runtime repo index for the deployed Bluepilot Builder dry-run path.

The first BP-137 live probe reached `POST /probe/dry-run`, but failed before orchestration could
continue because `builderScopeResolver` could not find `data/builder-repo-index.json`.

## Critical correction to external proposal

Claude's draft said the index should scan `builder/src`, but the live code still defaults to the
`soulmatch` target profile. The orchestrator uses target repo paths and later fetches contents from
`G-Dislioglu/soulmatch`.

Therefore the committed artifact must describe the current default target repo paths, for example
`server/src/lib/opusTaskOrchestrator.ts`, not Bluepilot-local `src/*.ts` paths. BP-138 corrects the
contract and implementation accordingly.
