# BP-140 Review Packet - Target-aware SmartPush

## Summary

BP-140 fixes the pre-Phase-B target-routing gap without enabling a real write.

Before this task, the real patch path in `builder/src/opusSmartPush.ts` always used
`G-Dislioglu/soulmatch`, even when the resolved Builder target profile was
`bluepilot-sandbox`. BP-140 threads the resolved `targetProfile.repo` from the orchestrator into
SmartPush, then into the direct GitHub patch call.

## Critical Safety Correction

The earlier BP-140 draft would have made the sandbox profile writable while SmartPush still targeted
Soulmatch. That would have made a "sandbox" real write land in the production Soulmatch repo.

This task does not do that. It only fixes target routing.

## Safety Properties

- No Maya kill-switch is opened or referenced.
- No target profile is changed.
- No endpoint is added.
- No live GitHub write is performed.
- Missing `targetRepo` still defaults to `G-Dislioglu/soulmatch`.
- Malformed `targetRepo` throws before gate, fetch, or write.
- For non-default repos, the legacy `/push` dispatch is fail-safe blocked instead of silently using
  the old Soulmatch path.
