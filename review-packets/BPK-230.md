# BPK-230 Review Packet - Permit Apply Main/Master Write Cap

## Scope

BPK-230 enforces hard cap #3 at the real write endpoint: `POST /probe/permit-apply` now fails closed before `smartPush` when the requested branch is `main` or `master`.

## Changes

- `builderSafetyPolicy.ts` accepts an optional `targetBranch` and classifies `main`/`master` as `class_3`, `manual_only`, blocked.
- `permitApply.ts` passes the request branch into `classifyBuilderTask(...)`.
- `permitApply.test.ts` proves `main` and `master` return 403 and never call `smartPush`.
- `builderSafetyPolicy.test.ts` proves protected branches are blocked while non-protected branches still allow the existing approved flow.

## Safety Notes

- No new route, proof-gate, permit system, local budget cap, emergency stop, provider behavior, DB behavior, package file, workflow, env variable, deploy, or runtime activation was added.
- Non-protected branches keep the existing permitApply behavior.
- A future explicit proof-gate can reopen main/master under a separate contract.
