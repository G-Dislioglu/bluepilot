## Ziel

BP-073 baut den ersten lokalen Live-Builder-Adapter-Readiness-Candidate Mock.

Der Mock prueft den BP-072 Handoff deterministisch und erzeugt ein lokales Live Builder Adapter Readiness Output Envelope.

## Was wurde geaendert?

- `tools/live-builder-adapter-readiness-candidate.cjs` angelegt.

- `contracts/BP-073.json` angelegt.

## Sicherheitsgrenzen

BP-073 macht nicht:

- kein Live Builder Call,
- kein Builder Task Create,
- keine Builder Adapter Konfiguration,
- keine Auth,
- keine Secrets,
- keine DB oder Persistenz,
- keine echte Approval-Wirkung,
- kein Approval Recording,
- kein Execute / Approve / Push / Deploy,
- keine Ziel-Datei-Writes.

`builder_task_create_allowed` bleibt immer `false`.

`builder_execute_allowed` bleibt immer `false`.

`live_builder_call_allowed` bleibt immer `false`.

## Erwartetes Verhalten

Der Live Builder Adapter Readiness Candidate Mock:

- gibt `live_builder_adapter_readiness_prepared` fuer vorbereitete Builder Task Create Readiness Candidates aus,
- gibt `requires_human_review` fuer Readiness Notes aus,
- gibt `blocked` fuer non-prepared Builder Task Create Readiness Status aus,
- gibt `blocked` fuer Live Builder Target, Auth, Secrets, Persistence oder Network Requests aus.
