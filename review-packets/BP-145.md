## What Changed

1. `CorridorAssessInput` now carries optional write-permit fields:
   `permitId`, `repo`, `branch`, `path`, `op`, `baseSha`, `contentHash`, and `contentLen`.
2. `putFileContent` keeps its existing no-mode behavior, but can now run in explicit
   create-only or update-only mode:
   - `create`: PUT without `sha`.
   - `update`: PUT with exactly `expectedBaseSha`.
3. The direct whole-file SmartPush path can use a `writePermit` option. It computes the shared
   content hash from final UTF-8 content before base64 encoding, asks the Maya corridor with the
   permit coordinates, and writes only when the corridor returns `reason: "permit_consumed"`.
4. Permit mode is intentionally narrow in this stage: exactly one whole-file create/update. Patch
   jobs are rejected before corridor or GitHub write calls.
