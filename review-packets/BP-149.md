# BP-149 Review Packet - Maya Sandbox Write Door

## Summary

BP-149 was a historical Bluepilot intermediate state before BP-150.

It replaced the older fixed permit-demo trigger with `POST /probe/sandbox-write`, an
env-gated route that could create, update, and delete files in
`G-Dislioglu/bluepilot-sandbox` on `main`.

This packet is a retrospective anchor reconciliation. It does not reintroduce the route.
BP-150 already retired it with HTTP 410 and removed the content write/delete body.

## Git Evidence

The BP-149 behavior is visible in commit `c7c3c97`:

- `STATE.md` and `docs/SESSION-LOG.md` contain explicit BP-149 entries.
- `builder/src/sandboxWrite.ts` implements `POST /probe/sandbox-write`.
- `builder/tests/sandboxWrite.test.ts` proves env gating, path/body validation,
  create/update behavior, and delete/undo behavior.
- `contracts/BP-149.json` and `review-packets/BP-149.md` were missing before this
  reconciliation.

The current `origin/main` state after BP-150 is `ba19c33`, where
`builder/src/sandboxWrite.ts` is a 410 stub and the old write/delete body is gone.

## Historical Behavior

The historical route:

- returned HTTP 403 unless `BLUEPILOT_SANDBOX_PERMIT_WRITE_ENABLED=true`;
- required `POST`;
- accepted only `path`, `contentBase64`, and optional `op`;
- validated the path syntactically but did not constrain it to `.bluepilot/`;
- targeted `G-Dislioglu/bluepilot-sandbox` on branch `main`;
- used GitHub Contents API state to choose create/update;
- supported `op: "delete"` for undo/delete when a target file existed;
- reported previous file state, including SHA and prior base64 content, in the response;
- did not require `permitId`, `writePermit`, or a Maya corridor consume.

## Critical Review

Claude's high-level drift call was correct: BP-149 was real work with missing
contract/review artifacts, not a case where `STATE.md` should simply be deleted.

One correction matters: the existing `STATE.md` and session log wording said the handler
accepted only `{ path, contentBase64 }`. The code in `c7c3c97` also accepted optional
`op: "delete"`. This reconciliation updates that wording so the anchor matches the code
that BP-150 retired.
