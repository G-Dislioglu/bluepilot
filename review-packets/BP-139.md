# BP-139 Review Packet - Sandbox write readiness

## Summary

BP-139 prepares Phase B without opening the Maya write kill-switch.

It adds:

- a dedicated `bluepilot-sandbox` Builder target profile,
- a guarded `POST /probe/sandbox-write-check` endpoint,
- tests proving the endpoint writes only to `G-Dislioglu/bluepilot-sandbox`, requires explicit
  confirmation, and never serializes token values.

## Critical safety decision

Claude's "open the kill-switch" request was intentionally not executed.

Code review confirmed:

- maya-core kill-switch is `MAYA_BUILDER_REPO_WRITE_ENABLED=true`,
- current Builder default target is still `G-Dislioglu/soulmatch`,
- `opusSmartPush.ts` is still hard-coded to `G-Dislioglu/soulmatch`.

Therefore opening the kill-switch now would be unsafe. BP-139 only proves that the deployed Builder
can write to the sandbox repo when explicitly asked.

## Probe behavior

Endpoint:

```text
POST /probe/sandbox-write-check
```

Required body:

```json
{"confirm":"write-to-bluepilot-sandbox"}
```

Behavior:

- reads token from `GITHUB_PAT`, `GITHUB_TOKEN`, or `GH_TOKEN`,
- creates a temporary file under `.bluepilot/` in `G-Dislioglu/bluepilot-sandbox`,
- deletes the same file,
- returns `writable`, `not_configured`, or `not_writable`,
- never returns token values,
- never accepts repo owner/name/path from user input.
