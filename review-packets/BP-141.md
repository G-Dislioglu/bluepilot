## Scope

BP-141 prepares the first real Builder write into `G-Dislioglu/bluepilot-sandbox` only. It does
not perform a live write and does not set any Render or Maya env values.

## Critical Corrections To Claude Draft

- The endpoint does not accept caller-provided instruction, target, scope, file, repo, branch, or dry-run mode.
- The endpoint always uses `.bluepilot/phase-b-real-write.md`.
- The endpoint does not set `skipDeploy:true`; in this orchestrator, that would skip the push phase.
- The endpoint sets `skipInlinePostPushChecks:true`, so it can push while avoiding irrelevant deploy/self-test checks for the sandbox repo.
- The existing BP-139 readiness probe was live-checked before build and returned `status:"writable"` with create/delete commits.

## Safety Properties

- Closed by default unless `BLUEPILOT_SANDBOX_REAL_WRITE_ENABLED === "true"`.
- Requires exact confirm phrase `real-write-to-bluepilot-sandbox`.
- Repo guard must resolve `bluepilot-sandbox` to `G-Dislioglu/bluepilot-sandbox`, branch `main`,
  `writePolicy:"sandbox_real_write"` and `pushAllowed:true`.
- The orchestrator call is fixed to `targetProfileId:"bluepilot-sandbox"`, `dryRun:false`,
  `scope:[".bluepilot/phase-b-real-write.md"]` and `targetFile:".bluepilot/phase-b-real-write.md"`.
- Maya corridor kill-switches remain external/manual and are not touched by code.
- No secret values are serialized in responses.
