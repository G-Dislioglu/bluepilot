# BP-135 Review Packet - Maya-Gate Readiness Probe

## Scope

BP-135 adds a browser-callable readiness probe for the Free Render `bluepilot-builder` service.
It does not add any Builder execution route and does not change the Maya gate client.

## Files Changed

| File | Reason |
|---|---|
| `contracts/BP-135.json` | WLP contract |
| `builder/src/health.ts` | Adds `/health/maya-gate` readiness logic |
| `builder/tests/health.test.ts` | Adds reachable/unreachable probe tests and token absence assertion |
| `docs/SESSION-LOG.md` | Anker log entry |
| `STATE.md` | Current state and next-step hints |
| `review-packets/BP-135.md` | Evidence packet |

## Route

```text
GET /health/maya-gate
```

The response includes:

```text
service
mayaCoreConfigured
timestamp
budget.reachable / budget.reason
corridor.reachable / corridor.reason
cost.reachable / cost.recorded / cost.reason
```

The response never includes the gate token or either token environment variable value.

## Safety Boundary

- No Builder task is started.
- No orchestrator, pipeline, smart-push, patch-mode, builder executor, or GitHub bridge module is imported.
- Corridor uses `actionKind: 'push'` with `dryRun: true`; it does not perform a push.
- Budget uses a tiny synthetic cheap-model payload.
- Cost uses the same tiny payload. This can create a small readiness cost record in maya-core;
  this is the only side effect and is explicitly treated as best-effort readiness evidence.

## Expected Live Check After Deploy

Open:

```text
https://bluepilot-builder.onrender.com/health/maya-gate
```

Expected if Render env binding is correct:

```text
budget.reachable = true
corridor.reachable = true
cost.recorded = true
```

If budget/corridor return `reachable: false` with `reason: gate_unavailable`, the token binding
or `MAYA_CORE_URL` is still wrong.

## Result

BP-135 gives us a stable, non-shell way to verify Bluepilot -> maya-core gate readiness on the
Render Free plan.
