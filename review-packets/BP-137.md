# BP-137 Review Packet - Phase-A Dry-Run Trigger

## Scope

BP-137 adds a narrow external trigger for the already migrated Builder orchestrator:

```text
POST /probe/dry-run
```

It forces `dryRun: true` and `skipDeploy: true` server-side.

## Correction To Draft

The Claude draft was useful but had two repo-state mismatches:

- `OpusTaskResult` does not expose `scopeFiles` at top level. BP-137 extracts them from the
  `scope` phase detail.
- A dry-run result returns before real SmartPush, so this endpoint does not re-prove external
  maya-core gate reachability. That proof is BP-135/BP-136. BP-137 reports the local Builder
  safety decision.

## Safety Boundary

- Request body cannot set `dryRun` or `skipDeploy`.
- The call site uses literal `dryRun: true` and `skipDeploy: true`.
- No package/dependency changes.
- No orchestrator, pipeline, scope resolver, gate client, provider, smart-push, patch-mode, or
  executor code changed.
- No token env var name appears in `builder/src/probeDryRun.ts`.
