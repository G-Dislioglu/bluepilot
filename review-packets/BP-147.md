## Deliberate Intermediate State

The permit path is built and live-proven once. It is not yet declared as the permanent
write policy for every future Builder write.

Required closed state after this proof:

- `bluepilot-builder`: `BLUEPILOT_SANDBOX_PERMIT_WRITE_ENABLED=false`
- `maya-core`: `MAYA_BUILDER_REPO_WRITE_ENABLED=false`
- `maya-core`: `MAYA_BUILDER_WRITE_PERMIT_ENFORCEMENT=false`
- if present: `MAYA_BUILDER_OPERATOR_APPROVED_WRITE=false`

Permanent enforcement is a later explicit decision, not a side effect of this proof.

## What This Proves

1. A permit can be issued by maya-core for exact repo/branch/path/op/content bytes.
2. Bluepilot can submit the exact content through the permit-gated sandbox write trigger.
3. The Maya corridor can consume the permit and allow one write.
4. The same permit cannot be reused.
5. The write window can be closed again, leaving no durable open write endpoint.
