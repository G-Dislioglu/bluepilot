## Ziel

BP-053 baut den ersten lokalen Human-Gate-Candidate Mock.

Der Mock prueft den BP-052 Handoff deterministisch und erzeugt ein lokales Human Gate Review Candidate Output Envelope.

## Sicherheitsgrenzen

BP-053 macht nicht:

- keine echte Approval-Aktion,
- kein Approval Recording,
- keinen Live Builder Call,
- kein Builder Task Create,
- kein Execute / Approve / Push / Deploy,
- keine Auth,
- keine Secrets,
- keine DB,
- keine Persistenz,
- keine Ziel-Datei-Writes.

`approval_effect` bleibt immer `none`.

`human_approval_recorded` bleibt immer `false`.

`builder_task_create_allowed` bleibt immer `false`.

`builder_execute_allowed` bleibt immer `false`.

## Erwartetes Verhalten

Der Human Gate Candidate Mock:

- gibt `review_candidate_prepared` fuer vorbereitete Builder Task Candidates aus,
- gibt `requires_human_review` fuer Risk Evidence aus,
- gibt `blocked` fuer non-prepared Builder Task Candidate Status aus,
- gibt `blocked` fuer wirksame Approval Effects aus.
