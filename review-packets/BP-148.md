# BP-148 Review Packet - Legacy Write Path Defuse

## Summary

BP-148 closes legacy write bypasses after the first permit-gated sandbox write was proven.

- `/probe/sandbox-real-write` is permanently retired and returns HTTP 410.
- `/probe/sandbox-write-check` remains as a diagnostic tool, but is closed by default behind
  `BLUEPILOT_SANDBOX_WRITE_CHECK_ENABLED=true`.
- `/probe/sandbox-permit-write` and the permit/policy path are unchanged.

This does not remove future Maya autonomy. The intended future path is that Maya may issue
one-shot write permits directly inside a narrow policy frame. BP-148 only removes old probe
endpoints that bypass that permit/policy path.

## Critical Review Of Input

Claude's recommendation was directionally correct: retire the old real-write probe and gate the
diagnostic write-check. Local verification added one important framing correction:

- The target is not "make Maya ask humans forever." The target is "no legacy write without an
  explicit gate." Later Maya self-writes should be unbureaucratic when policy allows them, but
  they must still be bound to one-shot permits and audit.

Local `rg` found no automated callers for `/probe/sandbox-write-check` in code or scripts. The
remaining references are contracts, documentation, and tests.
