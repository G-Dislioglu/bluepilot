# APP GOAL - Bluepilot

Erstellt: 2026-05-28
Geaendert: 2026-05-28 von Codex BP-085

## Was diese App im Kern leisten soll

Bluepilot ist ein eigenstaendiger Orchestrator fuer agentische Softwareentwicklung.
Ein Nutzer gibt ein Repo und ein Ziel vor; Maya koordiniert ein gebundenes Team von Codex-Agents unter WLP, Council Session, Evidence und menschlichem Gate.

Bluepilot baut keine unkontrollierte Autonomie. Jede Arbeit bleibt an Ziel, Scope, Review Packet und erlaubte Phase gebunden.

## Primary User Flow

1. Nutzer gibt Repo, Ziel und Risiko-/Scope-Grenzen vor.
2. Bluepilot erstellt eine Council Session und laedt nur erlaubten Kontext.
3. Maya weist Agents Tasks mit WLP-Contracts zu.
4. Agents arbeiten nach Phase und Scope, liefern Evidence und Review Packets.
5. Maya koordiniert Status, HARD STOPs, Context Deltas und Abschluss.
6. Nutzer gibt riskante Schritte, Merge, Deploy oder externe Wirkung final frei.

## Non-Goals

- Kein Soulmatch-Klon.
- Kein direkter AICOS-Write.
- Kein Live Builder, Task Create, Execute, Deploy oder Secret-Zugriff ohne eigenen Contract.
- Kein Ueberspringen der Build-Canvas-Reihenfolge.
- Kein Start paralleler Agents ohne aktiven Council Kernel.
- Keine UI-Integration bevor die benoetigten Vorphasen fertig sind.

## Erfolgsmassstab

- BP-C1 Council Kernel ist stabil, bevor BP-C2 startet.
- BP-C2 und BP-C3 parallelisieren nur innerhalb ihrer Phase.
- BP-C5 integriert fertige Vorarbeiten statt neue Grundfaehigkeiten isoliert zu bauen.
- Jeder Task hat Contract, Scope, Evidence, Review Packet und Reuse-Pfad.
- Kein Agent veraendert Dateien ausserhalb seines WLP-Scopes.
