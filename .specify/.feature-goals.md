# FEATURE GOALS - Bluepilot

Erstellt: 2026-05-28
Geaendert: 2026-05-28 von Codex BP-085
Quelle: `docs/BLUEPILOT_BUILD_CANVAS_DECISION_v0.md`

## Reihenfolge

Die Build-Reihenfolge ist absolut:

```text
BP-C1 Council Kernel
-> BP-C2 Multi-model Pool / Context Broker / WLP Tooling
-> BP-C3 Parallel Executor / Maya Memory / AICOS Auto-query
-> BP-C4 DiffLens / Browser Preview
-> BP-C5 Integration
```

BP-C2 startet erst, wenn BP-C1 vollstaendig gruen ist.

BP-C3 startet erst, wenn BP-C2 vollstaendig gruen ist.

BP-C4 startet erst, wenn BP-C3 vollstaendig gruen ist.

BP-C5 ist Integration und startet erst, wenn die benoetigten Vorphasen fertig sind.

## [BP-C1] - Council Kernel

Status: planned
Prioritaet: naechster sequentieller Build

### Goal

Maya kann eine Council Session lokal koordinieren und Agents ueber Dateien statt Polling steuern.

### Scope

- Maya Council Watcher.
- Council Agent Client.
- `.bluepilot/council/session.json`.
- `.bluepilot/council/agents/`.
- `.bluepilot/council/events.jsonl`.
- `.bluepilot/council/dedup.json`.
- Idempotency fuer doppelte Done-Events.
- HARD STOP Broadcast und Recovery-Regel.

### Acceptance

- [ ] Agent-Statusaenderung wird von Maya erkannt.
- [ ] Doppeltes Done-Event fuehrt zu keinem zweiten Assignment.
- [ ] HARD STOP pausiert Session und broadcastet an Agents.
- [ ] Events werden append-only dokumentiert.
- [ ] Kein BP-C2-Agent startet ohne aktiven Council Kernel.

### Hard Stop

- Zwei Besitzer schreiben dieselbe Council-Datei.
- Council Session kann HARD STOP nicht pausieren.
- Idempotency fehlt.

### Completion Signal

`FEATURE COMPLETE` + Feature Review Packet BP-C1.

## [BP-C2] - Parallel Foundation Phase

Status: blocked_until_BP-C1_complete
Prioritaet: nach BP-C1

### Goal

Drei phaseninterne Agents bauen die Grundlage fuer Modelle, Kontext und WLP-Werkzeuge parallel unter aktiver Council Session.

### Module

- Multi-model Pool: Provider Routing, BYOK, Eco Mode, Cost Gate.
- Context Broker: Repo Index, Architecture Digest, session-start in einem Call.
- WLP Tooling: Contract Schema, Verify-Haertung, Session Log.

### Acceptance

- [ ] Alle drei Module laufen unter Council Session.
- [ ] Kein Modul startet vor BP-C1.
- [ ] Context Broker liefert einen kontrollierten session-start.
- [ ] WLP Tooling bleibt kompatibel mit bestehenden Contracts.
- [ ] Multi-model Pool fuehrt keine echten Secrets ein.

### Hard Stop

- Start ohne aktiven Council Kernel.
- Secret-, Auth- oder DB-Zugriff ohne eigenen Contract.
- Parallelitaet ausserhalb der Phase.

### Completion Signal

`FEATURE COMPLETE` + Feature Review Packet BP-C2.

## [BP-C3] - Parallel Execution Phase

Status: blocked_until_BP-C2_complete
Prioritaet: nach BP-C2

### Goal

Bluepilot kann parallele Arbeit vorbereiten, Memory/Context sicher nutzen und AICOS read-only in Contracts referenzieren.

### Module

- Parallel Executor: Git Worktrees, Branches, Conflict Forecast.
- Maya Memory: Project Memory, User Preferences, proposal_only enforced.
- AICOS Auto-query: Embedding Match, Card Refs in Contract.

### Acceptance

- [ ] Drei Module laufen nur unter Council Session.
- [ ] Worktree-/Branch-Isolation ist lokal beweisbar.
- [ ] Maya Memory unterscheidet aktuelle Wahrheit, lokale Wahrheit und proposal_only.
- [ ] AICOS wird read-only genutzt.
- [ ] Card References landen in Contracts, keine Registry-Mutation.

### Hard Stop

- AICOS Write.
- Memory behauptet proposal_only als confirmed.
- Worktree-Konflikt ohne Forecast.

### Completion Signal

`FEATURE COMPLETE` + Feature Review Packet BP-C3.

## [BP-C4] - Visual Review Phase

Status: blocked_until_BP-C3_complete
Prioritaet: nach BP-C3

### Goal

Bluepilot macht Aenderungen visuell pruefbar, bevor Nutzer sie akzeptiert oder ablehnt.

### Module

- DiffLens: Visual Diff, Accept/Reject per Chunk, Evidence Diff, AICOS Warnings Panel.
- Browser Preview: Playwright/Puppeteer Screenshot, Live Page View, optional GOAT Bridge spaeter.

### Acceptance

- [ ] DiffLens zeigt nachvollziehbare Diffs.
- [ ] Browser Preview erzeugt lokale Evidence.
- [ ] UI-Review ersetzt kein Human Gate.
- [ ] Keine Deploy- oder Push-Wirkung ohne separaten Contract.

### Hard Stop

- Visuelle Preview behauptet echte Freigabe.
- Browser Preview braucht externe Secrets.
- Deploy/Push wird implizit aktiviert.

### Completion Signal

`FEATURE COMPLETE` + Feature Review Packet BP-C4.

## [BP-C5] - Integration Phase

Status: blocked_until_dependencies_complete
Prioritaet: nach BP-C4

### Goal

BP-C5 integriert fertige Vorphasen in ein nutzbares Produktbild.

### Module und Abhaengigkeiten

| Modul | Braucht vorher |
|---|---|
| Web UI | Context Broker |
| Remote Control | Council Kernel |
| GOAT Bridge | Parallel Executor |

### Acceptance

- [ ] Web UI startet erst mit brauchbarem Context Broker.
- [ ] Remote Control startet erst mit Council Kernel.
- [ ] GOAT Bridge startet erst mit Parallel Executor.
- [ ] BP-C5 baut keine fehlenden Grundfaehigkeiten neu.

### Hard Stop

- BP-C5 startet ohne benoetigte Vorphase.
- Integration wird als eigenstaendiger Build missverstanden.
- GOAT Bridge fuehrt lokale Git-/Terminal-Wirkung ohne eigenen Contract aus.

### Completion Signal

`FEATURE COMPLETE` + Feature Review Packet BP-C5.
